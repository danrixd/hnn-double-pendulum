# Data

Training data is generated by `data/generate.py`. Nothing in this folder is
checked into git — regenerate it with:

```bash
python -m data.generate --out data/double_pendulum.npz
```

The output archive contains:

| key            | shape              | description                                          |
|----------------|--------------------|------------------------------------------------------|
| `states`       | `(N, 4)` float64   | canonical states `[theta1, theta2, p1, p2]`          |
| `dstates`      | `(N, 4)` float64   | analytical `d/dt` of `states` (training targets)     |
| `trajectories` | `(T, S+1, 4)`      | per-trajectory rollouts used by the eval scripts     |
| `t`            | `(S+1,)` float64   | shared time grid                                     |
| `meta`         | JSON string        | generation parameters (n, dt, ranges, seed, params…) |

`N = T * (S + 1)` — every visited state along every rollout is a training
example. The analytical derivative is computed directly from Hamilton's
equations in `physics.dynamics`, so there is no finite-difference noise in the
supervision signal.
